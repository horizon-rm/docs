    RoboMaster 电控进阶培训：底盘与云台协同控制¶

    1. 前言¶

    在RM赛场上，机器人的灵活性至关重要。本教程将围绕 “底盘跟随云台” 、 “小陀螺行进” 以及 “云台控制” 三个核心场景展开。

    学习目标： 1. 理解底盘跟随中的PID解耦控制。 2. 掌握小陀螺模式下的坐标系转换与速度合成。 3. 区分云台控制的两种模式：普通双环角度PID与基于IMU的绝对姿态控制。

    2. 底盘跟随 (Chassis Follow)¶

    场景：云台瞄准目标时，底盘可以独立运动，但始终保证车头（底盘x轴）指向云台朝向，或者保持相对位置。

    2.1 原理¶

    底盘跟随的核心是角度闭环。我们需要控制底盘旋转，使得底盘的朝向（通常由陀螺仪yaw或底盘电机编码器推算）与云台的朝向之间的误差为0。

    设： - \( \theta_{gimbal} \)：云台当前朝向（相对于世界坐标系/惯性系）。 - \( \theta_{chassis} \)：底盘当前朝向。 - 误差 \( e = \theta_{gimbal} - \theta_{chassis} \)。

    通过PID控制器（通常是位置式PID）计算出一个角速度 \( \omega_{out} \)，叠加到底盘的运动学解算中。

    2.2 实现逻辑¶

    底盘运动通常由三部分组成：X速度、Y速度、旋转速度 \( \omega \)。

    2.3 关键点¶

    • 角度归一化：由于角度是周期性的（0°~360°或 -π~π），计算误差时必须进行归一化，防止突变。
    • 死区设置：当误差很小时（如<1°），停止转动，避免底盘微颤。

    3. 小陀螺行进 (Spin Move)¶

    场景：底盘一边高速旋转，一边进行平移运动。这使得敌方难以瞄准我们的装甲板。

    3.1 坐标系转换原理¶

    在小陀螺模式下，底盘相对于世界坐标系在旋转，但遥控器给出的移动指令通常是相对于底盘的（向前推摇杆，无论车头朝向哪，都朝前方移动），或者是相对于云台的（战术需求）。

    为了实现“小陀螺行进”，我们需要处理两个坐标系：

    1. 世界惯性系：云台稳定时所处的坐标系。
    2. 底盘坐标系：随底盘旋转的坐标系。

    如果我们要实现“车头朝北，但摇杆推右（底盘x轴正方向）时，车子往世界坐标系的东边平移”，则需要将遥控器的速度指令从底盘坐标系转换到世界坐标系，再计算电机的合成速度。

    3.2 数学公式¶

    设遥控器给定相对于底盘的速度 \( (v_{x\_chassis}, v_{y\_chassis}) \)，当前底盘朝向角为 \( \theta \)。 转换到世界系：

    \[ \begin{bmatrix} v_{x\_world} \\ v_{y\_world} \end{bmatrix} = \begin{bmatrix} \cos\theta & -\sin\theta \\ \sin\theta & \cos\theta \end{bmatrix} \begin{bmatrix} v_{x\_chassis} \\ v_{y\_chassis} \end{bmatrix} \]

    但在小陀螺模式下，我们通常希望运动轨迹与云台有关。更常用的策略是：将遥控器指令定义在云台坐标系下（因为云台是稳定瞄准的），然后通过底盘角度转换。

    目标： 车在旋转，但推摇杆，车总是朝着云台的正前方移动。

    转换公式（将云台系速度转世界系）：

    \[ \begin{bmatrix} v_{x\_world} \\ v_{y\_world} \end{bmatrix} = R(\theta_{gimbal}) \begin{bmatrix} v_{x\_remote} \\ v_{y\_remote} \end{bmatrix} \]

    其中 \( \theta_{gimbal} \) 是云台相对于世界的偏航角。

    接着，为了计算底盘电机的转速，需要将世界系速度转换到底盘系：

    \[ \begin{bmatrix} v_{x\_chassis\_cmd} \\ v_{y\_chassis\_cmd} \end{bmatrix} = R(-\theta_{chassis}) \begin{bmatrix} v_{x\_world} \\ v_{y\_world} \end{bmatrix} \]

    最后，\( \omega \) 单独由小陀螺旋转速度给定（恒定值），不参与坐标变换。

// 遥控器输入：remote_vx, remote_vy (云台坐标系下的期望移动方向)
// 小陀螺模式：给定一个固定的旋转速度 spin_omega (例如 300度/秒)

void spin_move_calc() {
    // 1. 遥控器指令 (云台系) -> 世界系
    float vx_world = remote_vx * cosf(gimbal_yaw) - remote_vy * sinf(gimbal_yaw);
    float vy_world = remote_vx * sinf(gimbal_yaw) + remote_vy * cosf(gimbal_yaw);

    // 2. 世界系 -> 底盘系 (因为电机控制需要底盘系下的速度)
    float vx_chassis_target = vx_world * cosf(chassis_yaw) + vy_world * sinf(chassis_yaw);
    float vy_chassis_target = -vx_world * sinf(chassis_yaw) + vy_world * cosf(chassis_yaw);

    // 3. 旋转速度直接赋值 (开环)
    float omega_target = spin_omega;

    // 4. 麦轮解算
    motor_speed[0] =  vx_chassis_target + vy_chassis_target + omega_target * wheelbase_factor;
    // ...
}

    4. 云台控制¶

    云台控制的目标是：让枪口稳定指向目标。这里分为两种模式。

    4.1 模式一：普通双环角度反馈¶

    应用场景：仅使用编码器作为位置反馈（无IMU或IMU失效时），或者用于俯仰轴（Pitch）控制。

    原理： - 内环（速度环）：控制电机转速，输出电流（转矩）。 - 外环（位置环）：控制电机到达指定角度，输出期望速度给内环。

    结构： 目标角度 -> [位置PID] -> 目标速度 -> [速度PID] -> 电流/电压 -> 电机 -> 实际角度/速度

float angle_pid_calc(float target_angle, float current_angle) {
    float error = normalize_angle(target_angle - current_angle);
    float speed_set = pid_calc(&pos_pid, error);
    // 限幅
    speed_set = clamp(speed_set, -MAX_SPEED, MAX_SPEED);
    return speed_set;
}

// 速度环 1ms 执行一次 (中断)
void speed_pid_loop() {
    float speed_set = target_speed_from_position_loop;
    float current_speed = get_motor_rpm();
    float current_out = pid_calc(&spd_pid, speed_set - current_speed);
    set_motor_current(current_out);
}

    4.2 模式二：基于IMU坐标系的云台控制¶

    应用场景：当底盘旋转时，云台需要通过IMU（陀螺仪）的角速度数据前馈或闭环，实现绝对的“稳像”或“指哪打哪”。

    在RM中，云台Yaw轴电机通常连接在底盘上。如果底盘转动，云台电机必须反向转动来补偿，才能保持枪口在世界坐标系中静止。

    4.2.1 绝对角度控制¶

    使用IMU解算出的欧拉角作为云台姿态的反馈量。

    设： - \( \theta_{yaw\_imu} \)：IMU测得的云台Yaw轴绝对角度。 - \( \theta_{yaw\_target} \)：目标绝对角度。

    4.2.2 惯性系前馈与扰动抑制¶

    为了抵抗底盘快速旋转带来的干扰，在速度环上添加IMU角速度前馈。

    底盘转动时，IMU检测到角速度 \( \omega_{disturb} \)。如果云台不动作，枪口会跟着转。为了稳定，云台电机应反向输出同样的角速度。

    公式： [ Output_{final} = Output_{PID} + K_{ff} \cdot \omega_{disturb} ] 其中 \( \omega_{disturb} \) 通常取IMU的Z轴角速度（Yaw轴扰动）。

    这种前馈可以显著提高云台在底盘高速旋转（小陀螺）时的稳定性。

    5. 总结与调试建议¶

    5.1 调试顺序¶

    1. 底盘基础：确保麦轮/全向轮的正逆运动学正确（推摇杆能斜向移动）。
    2. 云台单轴：先调Pitch轴（重力影响大，需要较大的积分或静力矩），再调Yaw轴。
    3. 底盘跟随：先降低PID的P值，观察跟随是否平滑，防止震荡。
    4. 小陀螺：先开环旋转，观察平移是否走直线；再闭环加入IMU补偿。

    5.2 常见问题¶

    • 底盘抖动：PID参数过大，或角度反馈噪声大（需要低通滤波）。
    • 云台掉头（缠绕）：Yaw轴通常有机械限位（如+-360度），代码中必须处理角度跳变，建议使用“累计角度”而非简单取模。
    • 小陀螺平移画圈：坐标变换中角度同步不及时（需要确保云台yaw角和底盘yaw角在同一时间戳下采样）。

    5.3 数学库准备¶

    建议在代码中封装好以下函数： - float normalize_angle(float angle)：将角度归一化到 [-π, π] 或 [-180, 180]。 - void rotate_vector(float angle, float vx, float vy, float *vx_out, float *vy_out)：向量旋转。
